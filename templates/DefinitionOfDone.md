# Definition of Done

## 1. Implementation is complete

- Code, configuration, and documentation changes match the agreed scope in the task.

## 2. Test checks pass

- Required tests pass.
- Lint and static checks pass.
- CI is green on required checks.

## 3. Review is complete

- Required approvals are collected.
- Review comments are resolved or explicitly documented with rationale.

## 4. Documentation is updated

- User-facing or operator-facing documentation is updated when applicable.
- Decision notes are captured when policy or process changed.

## 5. Operational readiness is confirmed

- Monitoring and logging impact reviewed when applicable.
- Rollback path validated for medium- and high-risk changes.

## 6. Release and accountability metadata exists

- Linked pull request, issue, and release note entry (if relevant) are present.
